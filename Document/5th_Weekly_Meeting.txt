<Ideal Final Result>

- 실사체가 아닐 것
- 플레이 타임이 짧을 것
  : 매칭으로 판짜서 하는 게임의 경우 플레이 타임이 20분 내외일 것
- 디자인 시 개성 또는 용도를 한눈에 알아볼 수 있게 할 것
- 고어 수준으로 잔인하지 않을 것
- 공포 게임은 지양
- 진입 장벽이 낮을 것
  : 초반에도 재미를 느낄 수 있게 해야 함. 동시에 중후반에도 재미를 느낄 수 있게 해야함. 
   초보자도 쉽게 재미를 느낄 수 있게 하기 위함,
   고인물들은 고인물들을 위한 이벤트로 환기시켜줄 것.
- 게임을 클리어(또는 만렙) 한 뒤에도 즐길 수 있는 컨텐츠가 존재할 것
- 시야가 좁고 어둡지 않을 것 (단, 공포게임이나 긴박한 분위기를 잡아야 하지 않을 경우에만)
- 많은 장르를 가져다 섞는 것은 좋으되, 장르마다 특성을 잘 살릴 것
- 플레이 방법이나 컨텐츠를 여러가지 만들 것
- 스토리가 없다면 명확한 목적성을 가질 것
- 카메라 움직임을 스프링처럼 이동해서 3D 멀미가 나지 않게 할 것(캐릭터 정수리에 붙어있을 경우에)
- 게임을 진행하는 데에 현질, 시간 등 부가적인 요소로 방해하지 말 것
- 게임 외에서 게임 내 필요한 정보를 얻게 하지 말 것
  : 예를 들어 해리포터 기반인 게임인데 해리포터의 코어팬이 아닌 이상 모르는 내용이 들어가 있는 상황이 있음
- 물리적인 면, 애니메이션에서 현실성이 반영 될 것
- 육성 요소가 있다면 교감(상호작용) 시스템을 넣을 것
- 플레이어가 현실적인 요소(물리적 위치와 사회적 위치)에 차별받지 않을 것
- 캐릭터, 지형 등 보기 불편할 정도의 극단적인 뷰는 필요에 의한 경우가 아니라면 지양할 것
 : 개성 및 캐릭터를 강조 하고 싶으면 낮은 뷰를 사용 할 것

애기 생존시키는 게임
목적: 플레이어보다는 애기를 생존시키는 것
배경: 인간이 아닌 플레이어가 유일한 애기를 기르는 것
플레이어: 각각 개성이랑 특징을 갖는 캐릭터 중 선택
         플레이어는 비교적 생존의 제약에서 자유로움(덜 먹어도 되고, 더 맞아도 되고)
애기: 생존의 제약이 많다(개복치 수준)
      행동 패턴은 딥러닝으로 학습 -> "어디서 어떻게 얻어..?" 플레이어랑 다른 행동 패턴인데..?
                                 -> 처음에는 단순한 행동 패턴을 보이다가 플레이하면서 플레이어의 행동 패턴을 따라해도 좋을 듯! (애가 자라는 느낌도 남~~~)
                                  => 생존 게임보다는 육성 게임으로 방향을 틀어도 괜찮을 듯 (육성, 약생존)
      호감도나 허기 같은 상태에 따라 행동이 달라짐

- 트렌드: 환경문제 -> 스토리나 게임 오브젝트 등에서 보여주기
         딥러닝 -> 애기 행동패턴 만들기 -> Unity 고려
         크로스드레싱 -> 애기의 성별을 불명확하게 하기

- 이상향: 캐릭터체 -> 캐릭터의 특성이나 개성을 보다 명확하게 보여주기 위해~
         스프링 카메라 -> 3인칭 일때 3D 멀미 예방
         진입 장벽 -> 캐릭터 특성으로 해결 (ex: 뫄뫄 캐릭터는 처음부터 최종 컨텐츠인 **을 할 수 있다)
         육성 요소로 아예 방향을 틀어서 교감과 호감도에 초점을 맞춰도 될 듯
